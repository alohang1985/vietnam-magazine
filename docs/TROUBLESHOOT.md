# TROUBLESHOOT — 장애 대응 체크리스트

1. 수집 실패
   - 원인: 사이트 접근 차단 / 페이월 / 네트워크 문제
   - 조치: 실패 URL 기록 및 스킵, 키워드 확장 재시도, editor_note에 경고 작성

2. 이미지 없음
   - 원인: API 키 누락 / 키 제한 / 검색 키워드 부적절
   - 조치: UNSPLASH/PEXELS 키 확인, 키워드 확장(지역명 + "landscape" 등), 수동 보충 요청

3. 품질 미달
   - 자동검증 실패 시: 자동 재생성(최대 2회) → 실패하면 draft로 저장 및 에디터 알림

4. API 장애
   - Strapi 다운: 서비스 재시작 (pm2 또는 systemd) / 로그 확인
   - 로깅: 모든 실패는 /logs/pipeline-errors.log에 타임스탬프와 원인 기록

5. 발행 실패
   - 원인: DB 잠금, Strapi 응답 오류, 네트워크
   - 조치: 재시도(지수 백오프) 후 실패 시 관리자 이메일/슬랙 알림

6. 중복 발행
   - 방지: 발행 락(레디스 락 또는 DB atomic flag) 사용 권장
